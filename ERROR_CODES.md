# エラーコード対応表

このドキュメントは、活動ローダー（activity-loader.js）で発生するエラーのコードと対処方法を記載しています。

---

## 概要

| エラーコード | 概要 |
|:------------|:-----|
| ERR-AL-001 | 設定エラー |
| ERR-AL-002 | 通信エラー |
| ERR-AL-003 | データ空 |
| ERR-AL-004 | DOM要素エラー |

---

## 詳細

### ERR-AL-001: 設定エラー

**原因**: Web App URLが設定されていません。

**対処方法**:
1. `js/activity-loader.js` を開く
2. 以下の行を見つける:
   ```javascript
   const WEB_APP_URL = 'YOUR_WEB_APP_URL_HERE';
   ```
3. `YOUR_WEB_APP_URL_HERE` を実際のGoogle Apps Script Web App URLに置き換える
4. ファイルを保存して再度読み込む

---

### ERR-AL-002: 通信エラー

**原因**: スプレッドシートからのデータ取得に失敗しました。

**考えられる原因と対処方法**:

| 原因 | 対処方法 |
|:-----|:---------|
| インターネット接続切断 | ネットワーク接続を確認 |
| Web App URLが誤っている | URLを再確認 |
| Apps Scriptのデプロイ設定エラー | 「アクセスできるユーザー」が「全員」になっているか確認 |
| Apps Scriptの権限エラー | Apps Scriptを再デプロイ |
| CORSエラー | ブラウザの開発者ツール(F12)→Consoleで詳細を確認 |

---

### ERR-AL-003: データ空

**原因**: スプレッドシートにデータがありません。

**対処方法**:
1. Googleスプレッドシートを開く
2. データが正しく入力されているか確認
3. 1行目がヘッダー（日付、タイトル、内容）になっているか確認
4. 2行目以降にデータがあるか確認
5. 日付列（A列）とタイトル列（B列）が空でないか確認

---

### ERR-AL-004: DOM要素エラー

**原因**: HTML内の `.activity-container` 要素が見つかりません。

**対処方法**:
1. `activity.html` を開く
2. `<div class="activity-container">` 要素が存在するか確認
3. クラス名のスペルミスがないか確認
4. JavaScriptの読み込み順序を確認（DOMより後にスクリプトを読み込む）

---

## 開発者向け情報

詳細なエラー情報はブラウザの開発者ツール（F12キー）→ Consoleタブで確認できます。

```
例: [ERR-AL-002] データ取得エラー: Failed to fetch
```
